# IT General Controls (ITGC) Audit Plan

## Overview

This repository contains a sample IT General Controls (ITGC) Audit Plan developed as part of my practical IT audit training. The document demonstrates my approach to planning an IT audit by identifying key risks, defining audit procedures, and specifying the evidence required to assess the design and operating effectiveness of IT controls.

## Audit Scope

The audit covers the following IT General Control (ITGC) domains:

- User Access Management
- Password Controls
- Backup Controls
- Change Management
- Antivirus Protection
- Logging & Monitoring

## Audit Objective

The objective of this audit is to evaluate whether the organization's IT General Controls are appropriately designed and operating effectively to protect information systems from:

- Unauthorized access
- Data loss
- System compromise
- Operational disruption

## Audit Methodology

For each control area, the audit plan includes:

- Control risks
- Audit procedures
- Evidence required
- Supporting documentation to validate control effectiveness

## Evidence Examples

The audit plan references evidence such as:

- Active Directory user listings
- HR employee and termination records
- Password and security policies
- Backup logs and restore test reports
- Change requests and CAB approvals
- Antivirus dashboard reports
- SIEM and Windows Event Logs
- Incident tickets

## Frameworks Referenced

- COBIT
- ISO/IEC 27001
- ITIL
- General ITGC Best Practices
